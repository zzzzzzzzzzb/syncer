# 局域网剪切板同步跨端页面设计（MVP）

## 1. 设计目标
- 让用户在 30 秒内理解并完成首次配对。
- 让用户在不学习网络概念的前提下感知“已连接、已加密、同步成功”。
- 让异常可恢复：断连、配对失败、冲突覆盖都有明确下一步动作。

## 2. 用户与场景
- 个人多设备用户：笔记本与手机之间快速共享文本。
- 小团队内网用户：同网段设备之间临时同步片段内容。
- 高安全敏感用户：关注“谁可同步、何时可撤销”。

## 3. 信息架构
- 一级导航
  - 配对
  - 设备
  - 同步记录
  - 设置
- 全局固定入口
  - 顶部状态条：网络状态、加密状态、最近同步时间
  - 全局通知：成功/失败 toast 与可重试操作

## 4. 全端页面清单
- P01 启动与权限引导页
- P02 首页（同步状态总览）
- P03 设备发现页
- P04 配对流程页（输入码/确认码/结果）
- P05 设备详情页（信任状态、会话状态、最后同步）
- P06 同步记录页（按时间倒序）
- P07 冲突处理页（保留本机/接受远端）
- P08 设置页（同步开关、自动启动、通知）
- P09 安全与信任管理页（已信任设备、撤销配对）
- P10 异常状态页（离线、网络抖动、握手失败）

## 5. 核心流程

### 5.1 首次使用流程
1. 进入 P01，完成剪切板权限与本地网络权限授权。
2. 自动进入 P03，展示局域网可发现设备。
3. 选择设备后进入 P04，输入 6 位配对码并确认。
4. 成功后跳转 P02，显示“已加密连接”与可同步状态。

### 5.2 日常同步流程
1. 用户复制文本触发本地同步事件。
2. P02 顶部状态从“同步中”切换为“同步成功”。
3. P06 追加一条记录，显示来源设备与时间。

### 5.3 异常恢复流程
1. 心跳超时触发 P10 横幅提示“连接中断”。
2. 用户可一键重连；失败则给出“重新配对”入口。
3. 若会话过期，直接引导到 P04 重建信任。

## 6. 分端布局规范

### 6.1 Windows / macOS（桌面）
- 导航形态：左侧固定侧边栏 + 右侧详情区。
- 首页布局：上方状态卡片 + 下方双栏（设备列表 / 最近同步）。
- 设备页增强：支持多选设备断开、批量信任撤销。
- 桌面增强能力
  - 系统托盘常驻，提供“暂停同步/快速重连”。
  - 快捷键入口：打开主窗口、跳转同步记录。

### 6.2 iOS / Android（移动）
- 导航形态：底部 Tab（首页、设备、记录、设置）。
- 首页布局：单列卡片流，突出“当前连接设备”与“一键配对”。
- 配对流程：使用分步式全屏页面，减少输入负担。
- 移动增强能力
  - 后台同步状态受系统策略影响，明确展示“前台活跃/后台受限”。
  - 推送通知可直接跳转到失败记录并执行重试。

### 7.3 P04 配对流程页
- Step 1：选择目标设备（来自发现列表）。
- Step 2：输入 6 位配对码，实时校验格式。
- Step 3：展示双方设备指纹后确认绑定。
- Step 4：结果页提供“去首页”与“查看设备详情”。

### 7.4 P09 安全与信任管理页
- 已信任设备列表：设备名、首次配对时间、最近活跃时间。
- 操作项：撤销信任、重新配对、查看指纹。
- 风险提示：撤销后该设备不可接收或推送同步事件。

## 8. 状态与反馈规范
- 同步状态：空闲 / 同步中 / 成功 / 失败 / 冲突待处理。
- 反馈层级
  - 轻量成功：toast 自动消失。
  - 可恢复失败：横幅 + 重试按钮。
  - 安全相关失败：阻断弹窗 + 明确风险文案。
- 文案原则
  - 使用用户语言，不暴露底层协议术语。
  - 对每个失败状态都给一个主操作按钮。

## 9. 交互与可用性要求
- 首次配对主路径不超过 4 步。
- 任何页面均可 2 次点击内到达“重新配对”入口。
- 关键按钮最小触控尺寸
  - 移动端：44x44
  - 桌面端：32x32
- 颜色语义
  - 成功：绿色
  - 警告：橙色
  - 错误：红色
  - 安全锁态：蓝色

## 10. 与 Rust Core 事件映射
- 发现事件 -> 更新 P03 设备列表与在线状态。
- 配对状态事件 -> 驱动 P04 步骤机与结果页。
- ClipboardUpdate/Ack -> 更新 P02 状态卡与 P06 记录流。
- 心跳超时/重连事件 -> 触发 P10 异常提示与恢复动作。

## 11. 交付物（供开发直接落地）
- 页面清单与导航结构：可直接建立 Flutter 路由表。
- 每页关键组件：可直接拆分 widget 层级。
- 状态映射规则：可直接绑定 `syncer-ffi` 事件流。
- 异常与安全交互：可直接对接重试与撤销接口。
